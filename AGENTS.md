# Agent Instructions

This repository uses a set of agent rules located in the `agents/` directory to guide development.

## Usage

- **For AI Agents**: When working on this codebase, you MUST read the relevant files in the `agents/` directory before making changes.
- **For Developers**: Refer to these documents for standards and workflows.

## Directory Structure

- `agents/branding.md`: Design system, colors, typography.
- **`agents/architecture.md`**: Tech stack, component structure, state management.
- **`agents/testing.md`**: TDD rules, testing strategies.
- **`agents/cicd.md`**: CI/CD pipeline and deployment rules.
- **`agents/git_repo.md`**: Branching and PR rules.
- **`agents/code.md`**: Coding standards (Svelte 5).
- **`agents/dev.md`**: Developer commands and workflow.

**Note**: The instructions in these files are authoritative. Deviations require explicit approval.
